Mapping: Database Tables → App Components

bioinformatics / clinical_bioinfo_sheet / discovery_bioinfo_sheet

Backend routes: GET|PUT|DELETE /api/bioinformatics (generic), GET /api/bioinfo/discovery, GET /api/bioinfo/clinical
Storage / services: storage.getLabProcessingQueue(), storage.updateLabProcessing(), storage.deleteLabProcessing() (labProcessing rows are surfaced as BI records)
Frontend: Bioinformatics.tsx
Notes: adapters read/write lab_processing rows, map fields between labProcessing ↔ bioinfo sheet shape.
clients

Backend routes: GET /api/clients, POST /api/clients
Storage / services: storage.getClients(), storage.createClient()
Frontend: used by admin/lead UI (no dedicated Clients.tsx — surfaced in admin/lead pages / imports)
Notes: client records are referenced by leads and finance workflows.
finance / finance_records / finance_sheet

Backend routes: GET /api/finance/records, POST /api/finance/records, DELETE /api/finance/records/:id, GET /api/finance/stats, GET /api/finance/pending-approvals and adapters: GET|POST|PUT|DELETE /api/finance-sheet
Storage / services: storage.getFinanceRecords(), createFinanceRecord(), updateFinanceRecord(), deleteFinanceRecord()
Frontend: FinanceManagement.tsx (and UI libs used by finance module)
Notes: two-layer access: typed storage methods + raw finance_sheet adapter to read legacy sheets.
gc_registration / genetic_counselling / genetic_counselling_records

Backend routes: GET|POST|PUT|DELETE /api/genetic-counselling, adapters: GET|POST|PUT|DELETE /api/gc-registration
Storage / services: storage.getGeneticCounselling(), createGeneticCounselling(), updateGeneticCounselling(), deleteGeneticCounselling()
Frontend: GeneticCounselling.tsx
Notes: GC flows are created on lead→sample conversion and notified via notificationService.
lab_process_clinical_sheet / lab_process_discovery_sheet / lab_processing

Backend routes: GET|POST|PUT|DELETE /api/lab-processing, GET /api/lab-process/discovery, GET /api/lab-process/clinical
Storage / services: storage.createLabProcessing(), getLabProcessingQueue(), updateLabProcessing(), deleteLabProcessing()
Frontend: LabProcessing.tsx
Notes: lab processing rows drive both lab UI and bioinformatics adapters; notifications sent when status changes.
lead_trfs

Backend routes: POST /api/uploads/trf-db (insert blob), GET /api/uploads/trf/:id (download)
Storage / services: storage.createLeadTrf(), storage.getLeadTrf() (uses lead_trfs table)
Frontend: TRF upload/download triggered from LeadManagement.tsx (or lead detail UI)
Notes: BLOB stored as base64 in DB; also POST /api/uploads/trf (file-system upload) exists.
leads

Storage / services: storage.createLead(), getLeads(), getLeadById(), updateLead(), updateLeadStatus(), convertLead(), deleteLead()
Frontend: LeadManagement.tsx

lims_users / users

notifications
